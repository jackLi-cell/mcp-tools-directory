Claude Desktop MCP Configuration Guide

This guide helps you configure MCP Server in Claude Desktop from scratch. Once complete, Claude will be able to call external tools for file operations, web searches, and other tasks.

Prerequisites

  • Claude Desktop installed (macOS or Windows)
  • Node.js 18 or higher installed (for running Node.js-based MCP Servers)
  • Basic command-line knowledge

Check if Node.js is installed:

node --version
# Should output v18.x.x or higher

Configuration Steps

Step 1: Locate the Configuration File

Claude Desktop MCP configuration file location:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

If the file does not exist, create it manually.

Step 2: Edit the Configuration File

Open the config file and write the following (using filesystem MCP Server as an example):

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Documents"
      ]
    }
  }
}

Replace /Users/yourname/Documents with the directory path you want Claude to access.

Step 3: Restart Claude Desktop

Completely quit Claude Desktop (not just minimize), then reopen it. On macOS, right-click the menu bar icon and select "Quit".

Step 4: Verify Connection

After restarting, you should see a hammer icon (tool icon) on the left side of the chat input. Click it to view connected MCP Servers and available tools.

Adding Multiple MCP Servers

Add multiple key-value pairs in the mcpServers object:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/yourname/Documents"]
    },
    "fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}

Viewing Logs for Troubleshooting

If MCP Server is not connecting properly, check the log files:

  • macOS: ~/Library/Logs/Claude/mcp*.log
  • Windows: %APPDATA%\Claude\logs\mcp*.log

The logs will show the specific reason for Server startup failure.

Important Notes

  • The config file must be valid JSON format. Extra commas or missing quotes will cause parse failures.
  • Backslashes in paths (Windows) need to be escaped as double backslashes, e.g., "C:\\Users\\name\\Documents".
  • You need to fully restart Claude Desktop after each config change.
  • Do not commit config files containing API keys to public repositories.

FAQ

What if the tool icon is not visible after configuration?
Confirm you fully restarted (not just refreshed the window). Check if the config file JSON format is correct using cat config.json | python -m json.tool. Check log files for specific error messages.
What if npx command is not found?
Confirm Node.js is properly installed and in system PATH. Run which npx (macOS) or where npx (Windows) in terminal to confirm the path. If using nvm, you may need to write the full path of npx in the config.
How many MCP Servers can be connected simultaneously?
No hard limit, but each Server is an independent process and too many will consume system resources. Only enable Servers you currently need.
