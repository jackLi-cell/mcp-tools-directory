How to Store API Keys and Environment Variables

许多 MCP Server requires API Key 或 Token 才能工作。本文介绍如何安全存储这些凭据，避免意外泄露到公开仓库或日志medium。

Core Principles

  • Never hardcode:API Key 不应出现在代码文件或公开的配置文件medium
  • Never commit:Files containing real keys should not be tracked by git
  • Least privilege:Create separate keys for each use case, granting only necessary permissions
  • Regular rotation:Rotate keys regularly, especially immediately when a leak is suspected

Method 1: Write Directly in MCP Config env Field

The simplest approach, suitable for personal use when config files will not be committed to public repositories:

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
      }
    }
  }
}

注意：Ensure this config file is added to .gitignore and will not be committed to version control.

Method 2: Use .env File + Environment Variable References

将 Key 存放在单独的 .env 文件medium，配置文件引用环境变量名：

第一步：Create a .env file:

# .env file (do not commit to git)
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
OPENAI_API_KEY=sk-xxxxxxxxxxxx

第二步：Add .env to .gitignore:

# .gitignore
.env
.env.local
.env.*.local

第三步：Load environment variables before starting clients (macOS/Linux):

# 在 ~/.zshrc 或 ~/.bashrc medium添加
export GITHUB_TOKEN="ghp_xxxxxxxxxxxx"

Claude Desktop 会继承系统环境变量，因此 env Fieldmedium可以引用已设置的变量。

Method 3: Use System Key Management Tools

对安全性要求更High的场景，可以使用操作系统的密钥管理：

macOS Keychain

# Store
security add-generic-password -a "mcp" -s "github-token" -w "ghp_xxxxxxxxxxxx"

# 读取（可在脚本medium使用）
security find-generic-password -a "mcp" -s "github-token" -w

Windows Credential Manager

Add generic credentials via "Control Panel > Credential Manager", or use PowerShell:

# Requires CredentialManager module
Install-Module -Name CredentialManager
New-StoredCredential -Target "mcp-github" -UserName "token" -Password "ghp_xxxxxxxxxxxx"

Emergency Response After a Leak

  1. Rotate immediately:Immediately revoke the old key and generate a new one on the corresponding platform (GitHub, OpenAI, etc.)
  2. Check usage logs:Review the key call logs to confirm if there was any abnormal usage
  3. 清理 git 历史：如果 Key 被提交到 git，仅删除文件不够，requires清理历史记录
  4. 更新配置：用新 Key 更新所有使用该凭据的配置

重要：即使仓库Yes私有的，一旦 Key 进入 git 历史就应视为已泄露。GitHub 等平台会扫描公开仓库medium的 Key 并自动通知。

Important Notes

  • 不同服务的 Key 格式不同：GitHub 以 ghp_ 开头，OpenAI 以 sk- 开头。
  • 创建 Key 时选择最小Permission Scope。例如 GitHub Token 只勾选 repo:read 而非全部权限。
  • 设置 Key 的过期时间，避免永久有效的 Key 被遗忘后泄露。
  • 团队协作时，每人使用自己的 Key，不要共享同一个。

FAQ

Claude Desktop 的配置文件requiresAdd to .gitignore 吗？
Claude Desktop 的配置文件在系统应用数据目录medium（不在项目目录），通常不会被 git 跟踪。但如果你把配置文件Copy到了项目medium做备份，那个副本requiresAdd to .gitignore。
env Fieldmedium的值会出现在日志medium吗？
正常情况下不会。但某些 Server 的调试日志可能打印环境变量，建议不要在生产环境开启 Server 的 debug 模式。
可以多个 Server 共用同一个 API Key 吗？
技术上可以，但不建议。为每个 Server 创建独立的 Key，这样某个 Key 泄露时只requires轮换一个，不影响其他 Server。
