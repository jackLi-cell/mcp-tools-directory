MCP Permission Risk Checklist

MCP Server runs as a local process with the system permissions you grant. Before connecting any MCP Server, confirm each item in this checklist to reduce security risks.

Pre-Connection Checklist

Check ItemdescriptionRisk
Source CredibilityIs it from an official repository or well-known developer? How are the star count and maintenance frequency?High
Permission ScopeWhat tools does the Server declare? Does it exceed your actual needs?High
File System AccessDoes it need to read/write files? Is the access scope limited to specific directories?medium-High
Network AccessDoes it need external network access? Which domains does it connect to?medium
Command ExecutionCan it execute Shell commands? Is there a command whitelist restriction?High
API Key Requirementsrequires哪些凭据？这些凭据的Permission ScopeYes什么？medium
Data TransferWill it send local data to external services?High

Permission Scope评估

Low-Risk Characteristics

  • Read-only operations (e.g., weather queries, document search)
  • Does not require API Key
  • Does not access file system or only accesses specified directories
  • Does not execute Shell commands
  • Open source with active community review

High-Risk Characteristics

  • Can execute arbitrary Shell commands
  • Accesses entire file system (root directory /)
  • Requires high-privilege API keys (e.g., GitHub repo write access)
  • Can send network requests to any domain
  • Closed source or lacks community review
  • Can modify system configuration or install software

凭据保护

  • API Key 只在配置文件的 env Fieldmedium设置，不要硬编码在其他地方
  • 为每个服务创建专用的 API Key，使用最小权限原则
  • 定期轮换 Key，尤其Yes在怀疑泄露时
  • 不要把包含 Key 的配置文件提交到公开仓库
  • 将配置文件路径加入 .gitignore
# .gitignore
.cursor/mcp.json
claude_desktop_config.json
.env

企业环境额外检查

  • 连接公司内部系统前，需经过团队安全审核
  • 不要将生产数据库的凭据配置到 MCP Server
  • 使用只读账号连接数据库类 Server
  • 确认 Server 不会将公司数据发送到外部
  • 在沙箱环境medium先测试，确认行为符合预期后再用于正式环境

Important Notes

  • MCP Server 的权限等同于你的用户权限，它能做的事你都能做，反之亦然。
  • AI 模型可能在对话medium被诱导调用危险工具，限制 Server 的Permission ScopeYes最有效的防护。
  • 定期检查已安装的 Server 列表，移除不再使用的。
  • 关注 Server 仓库的安全公告和版本更新。

FAQ

如何限制 filesystem Server 的访问范围？
在 args medium只指定requires访问的目录路径，不要指定根目录。例如 "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/name/projects"] 只允许访问 projects 目录。
第三方 Is MCP Server safe?
取决于具体项目。优先选择官方维护的 Server（@modelcontextprotocol 组织下的包）。第三方 Server 建议检查源码、Star 数、最近更新时间和 Issue mediumwhether有安全相关讨论。
发现 MCP Server 有安全问题怎么办？
立即从配置medium移除该 Server 并重启client。如果涉及 API Key，立即轮换相关凭据。向 Server 仓库提交安全 Issue。
