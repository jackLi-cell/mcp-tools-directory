Cursor MCP Configuration Guide

Cursor is an AI-powered code editor that supports connecting external tools via MCP protocol. This guide explains how to configure MCP Server in Cursor.

Prerequisites

  • Cursor editor installed (version 0.45 or higher)
  • Node.js 18+ installed (for npx-based Servers) or Python 3.10+ (for uvx-based Servers)

Configuration Steps

Step 1: Open MCP Settings

In Cursor, access MCP configuration through:

  1. Open Settings (shortcut Cmd+, or Ctrl+,)
  2. Find "MCP" option in the left menu
  3. Click the "Add new MCP server" button

You can also directly edit the .cursor/mcp.json file in the project root.

Step 2: Edit Configuration File

Cursor MCP configuration file locations:

  • Project level:.cursor/mcp.json(in project root)
  • Global level:~/.cursor/mcp.json(in user directory)

Configuration file format example:

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/path/to/your/project"
      ]
    }
  }
}

Step 3: Verify Connection

After saving the configuration, Cursor will automatically attempt to connect to the MCP Server. Connected Servers will show a green status indicator in the MCP settings page.

If it shows red or yellow, click the Server name to view error details.

Using in Agent Mode

MCP tools are available in Cursor's Agent mode (Composer Agent):

  1. Open Composer (Cmd+I or Ctrl+I)
  2. Switch to Agent mode
  3. AI will automatically call connected MCP tools as needed

Note: Regular Chat mode and Edit mode do not support MCP tool calls.

Important Notes

  • Project-level config (.cursor/mcp.json) will be committed to version control. Do not write API keys in it.
  • For Servers requiring API keys, use environment variables. Store keys in a .env file and add it to .gitignore.
  • Cursor's MCP support is still rapidly evolving. Configuration format may change with version updates.
  • If a Server fails to start, try manually running the command + args in terminal to verify the command works.

FAQ

What is the difference between project-level and global configuration?
Project-level config only applies to that project, suitable for project-specific tools (like database connections). Global config applies to all projects, suitable for general tools (like file search, web scraping). When both exist, they are merged.
What to do when MCP tools are not responding in Cursor?
Confirm you are using Agent mode (not regular Chat). Check if the Server status is green in MCP settings. Try clicking the "Restart" button to restart the Server.
Can I use Claude Desktop's MCP config in Cursor?
They cannot be shared directly. The config file paths and formats differ slightly and need separate configuration. However, MCP Servers themselves are universal - the same Server can be used by both clients simultaneously.
